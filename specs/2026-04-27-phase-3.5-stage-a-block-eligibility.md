# Phase 3.5 Stage A -- block-eligibility full coverage

## Goal

Wire the remaining 6 block-eligibility keywords (MENACE, UNBLOCKABLE,
SHADOW, HORSEMANSHIP, FEAR, INTIMIDATE) into match-runner
`_resolve_combat`. Phase 3 partial covered FLYING + REACH only; Stage A
extends to all evasion/blocking restrictions.

Stage A's data-structure refactor (assignments dict from
`id(atk) -> blk` to `id(atk) -> list[blk]`) is required for Stage B's
blocker-side lifelink iteration -- this stage is the foundation for
the rest of Phase 3.5.

## Scope

### In scope
- KWTag enum already has all 6 keywords -- no additions needed
- `_legal_blockers(atk, blockers)` helper: per-attacker list of legal
  blockers based on evasion restrictions
- `_resolve_combat` blocker-assignment loop refactored to handle
  multi-blocker case (menace requires 2+, unblockable allows 0)
- `assignments` data structure: `id(atk) -> list[blk]` (was single `blk`)
- `_resolve_strike_step` updated to iterate the blocker list per
  attacker (sums damage across all blockers, deathtouch applies to
  each blocker individually, lifelink accumulates per blocker contact)

### Explicitly out of scope
- Defender, Haste fix, blocker-side lifelink -- Stage B per
  Phase 3.5 staging spec
- Protection cluster (hexproof/ward/protection/shroud) -- Stage C
- Additive flying-color variants ("Flying except by red creatures")
  -- not standard MTG, no real-world cards
- Compound evasion ("Can only be blocked by 2+ creatures with flying")
  -- rare, defer to Annotated Imperfections if encountered

### Step 2 -- implement `_legal_blockers` helper (~10 min)

Add to `engine/match_runner.py` between `_safe_toughness` (line 448)
and `_resolve_combat` (line 461):

```python
def _legal_blockers(atk, blockers):
    """Filter blockers list to those legally able to block `atk` per
    evasion keywords. Returns a new list (does not mutate input).

    Phase 3.5 Stage A (2026-04-27): full block-eligibility coverage.
    Pre-Phase-3.5: only FLYING + REACH handled inline.
    """
    from engine.keywords import KWTag

    if KWTag.UNBLOCKABLE in atk.tags:
        return []  # Cannot be blocked at all

    legal = list(blockers)

    if KWTag.FLYING in atk.tags:
        legal = [b for b in legal
                 if KWTag.FLYING in b.tags or KWTag.REACH in b.tags]

    if KWTag.SHADOW in atk.tags:
        # Only shadow blockers
        legal = [b for b in legal if KWTag.SHADOW in b.tags]
    else:
        # Non-shadow attackers can't be blocked by shadow creatures
        legal = [b for b in legal if KWTag.SHADOW not in b.tags]

    if KWTag.HORSEMANSHIP in atk.tags:
        legal = [b for b in legal if KWTag.HORSEMANSHIP in b.tags]

    if KWTag.FEAR in atk.tags:
        # Fear: only artifact creatures or black creatures can block
        legal = [b for b in legal
                 if 'Artifact' in (b.type_line or '')
                 or 'B' in (b.colors or [])]

    if KWTag.INTIMIDATE in atk.tags:
        # Intimidate: only artifact creatures or creatures sharing
        # a color with attacker can block
        atk_colors = set(atk.colors or [])
        legal = [b for b in legal
                 if 'Artifact' in (b.type_line or '')
                 or set(b.colors or []) & atk_colors]

    return legal
```

### Step 3 -- refactor blocker assignment in `_resolve_combat` (~15 min)

Replace single-blocker assignment with list-of-blockers + menace
handling. Current code (lines ~496-510):

```python
# Blocker assignment: biggest legal blocker -> biggest attacker.
# Flying attackers can only be blocked by flying or reach blockers.
available_blockers = list(blk_sorted)
assignments = {}  # id(attacker) -> blocker
for atk in atk_sorted:
    atk_flies = KWTag.FLYING in atk.tags
    for i, blk in enumerate(available_blockers):
        if atk_flies and (KWTag.FLYING not in blk.tags
                          and KWTag.REACH not in blk.tags):
            continue
        assignments[id(atk)] = blk
        available_blockers.pop(i)
        break
```

Replace with:

```python
# Blocker assignment (Phase 3.5 Stage A): full evasion coverage.
# Each attacker may be assigned a LIST of blockers (menace requires 2+).
# Defender prioritizes blocking biggest attackers first; uses biggest
# legal blockers; for menace attackers, assigns 2 blockers if available
# else assigns none (so menace attacker goes unblocked).
available_blockers = list(blk_sorted)
assignments = {}  # id(attacker) -> list[blocker]
for atk in atk_sorted:
    legal = _legal_blockers(atk, available_blockers)
    if not legal:
        assignments[id(atk)] = []
        continue
    needed = 2 if KWTag.MENACE in atk.tags else 1
    if len(legal) < needed:
        # Can't satisfy menace requirement -- attacker goes unblocked
        assignments[id(atk)] = []
        continue
    # Pick `needed` biggest legal blockers
    chosen = legal[:needed]
    assignments[id(atk)] = chosen
    for b in chosen:
        available_blockers.remove(b)
```

### Step 4 -- update `_resolve_strike_step` for blocker lists (~15 min)

Current single-blocker logic (lines ~520-575) iterates `blk = assignments.get(id(atk))`. Refactor to iterate the list:

```python
def _resolve_strike_step(strike_attackers):
    """Resolve damage for one strike step (first-strike or regular).
    Updates atk_dead, blk_dead, total_dmg, lifelink_gain in enclosing
    scope. Skips attackers/blockers already dead from prior step."""
    nonlocal total_dmg, lifelink_gain
    for atk in strike_attackers:
        if id(atk) in atk_dead:
            continue
        atk_pwr = _safe_power(atk)
        atk_tou = _safe_toughness(atk)
        atk_dt  = KWTag.DEATHTOUCH in atk.tags
        atk_ll  = KWTag.LIFELINK in atk.tags
        atk_tr  = KWTag.TRAMPLE in atk.tags

        blocker_list = assignments.get(id(atk), [])
        # Filter out blockers already dead from prior step
        live_blockers = [b for b in blocker_list if id(b) not in blk_dead]

        if not live_blockers:
            # Unblocked: damage to defending player; lifelink gains us life
            total_dmg += atk_pwr
            if atk_ll:
                lifelink_gain += atk_pwr
            continue

        # Blocked combat -- attacker assigns damage among blockers in
        # order (real Magic CR 509.4: attacker chooses order; we use
        # the natural list order from assignment, which is biggest-first).
        damage_remaining = atk_pwr
        for blk in live_blockers:
            blk_pwr = _safe_power(blk)
            blk_tou = _safe_toughness(blk)

            # Blocker damage to attacker (each live blocker contributes)
            if blk_pwr > 0 and not _is_indestructible(atk):
                if blk_pwr >= atk_tou or KWTag.DEATHTOUCH in blk.tags:
                    atk_dead.add(id(atk))

            # Attacker damage to this blocker
            if damage_remaining > 0:
                # Deathtouch: 1 damage is lethal
                damage_to_assign = 1 if atk_dt else min(damage_remaining, blk_tou)
                if not _is_indestructible(blk):
                    if damage_to_assign >= blk_tou or atk_dt:
                        blk_dead.add(id(blk))
                if atk_ll:
                    lifelink_gain += damage_to_assign
                damage_remaining -= damage_to_assign

        # Trample: excess damage after all blockers go to defending player
        if atk_tr and damage_remaining > 0:
            total_dmg += damage_remaining
            if atk_ll:
                lifelink_gain += damage_remaining
```

## Mid-execution amendments

### Amendment 1: multi-blocker damage accumulation bug (2026-04-27)

**Trigger:** Stage A first-pass implementation had per-blocker
death-check on the attacker — attacker died only if a SINGLE blocker
had lethal power. Per CR 510.1, blocker damage SUMS on the attacker
before lethality check. Two 2/2s vs a 3/4 menace attacker should kill
it (4 >= 4); pre-fix code kept it alive (neither blocker had 2 >= 4).

**Hidden in BE/Murktide mirrors** because neither deck has menace
creatures, so multi-blocker assignment never fires and the per-blocker
loop runs once (matching pre-Stage-A single-blocker behavior).

**Fix:** restructured `_resolve_strike_step` per-attacker block into
two sequential passes over `live_blockers`:
- Pass 1: accumulate `total_damage_to_atk` + `any_blocker_deathtouch`
  flag, THEN apply combined damage to attacker (death check)
- Pass 2: existing per-blocker damage-assignment-from-attacker
  (lifelink + trample math unchanged)

Blocker-side LIFELINK accumulation point also moved into Pass 1 (still
no-op `pass` placeholder; Stage B activates the defender_lifelink_gain
return-tuple threading).
