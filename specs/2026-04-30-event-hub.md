# Spec: Event Hub

## Goal

Upgrade the basic event search tab into a full competitive event management tool.
A competitive player needs to discover, plan, track, and reflect on their tournament
schedule — not just search once and forget. The Event Hub provides all of this in one place.

## Context

Current state (2026-04-30): `gui/tabs/event_finder_tab.py` provides search + results table.
Backend: `scrapers/event_finder.py` → Wizards GraphQL API. Works well for discovery but
has no persistence, no planning, no personal history.

Design research: interviewed the full feature space (ultrathink session). Five dimensions
identified: discovery, planning, tracking, store relationships, post-event analysis.

## Scope by Session

### Session 1 (THIS SESSION — build now)

1. **DB tables** — `event_bookmarks`, `store_bookmarks` in `mtg_meta.db`
2. **EventHubTab** — 4-view container replacing EventFinderTab
3. **Search view** — refactored from EventFinderTab + bookmark button per row
4. **Calendar view** — monthly grid, colored event chips, prev/next navigation
5. **My Events view** — bookmarked events with status / notes / result fields
6. **My Stores view** — saved stores with quick-filter
7. **.ics export** — export bookmarked events to standard calendar file

### Session 2 (next session)

8. **RC prep countdown banner** — reads RC date from preferences, shows days remaining
9. **Drive time estimation** — heuristic or OpenRouteService free tier
10. **Conflict detection** — warn when two bookmarked events share a date
11. **Post-event Spicerack enrichment** — after event date passes, pull top 8 via Spicerack

### Session 3 (later)

12. **Format health dashboard** — events per format in your area over 30 days
13. **Team events view** — shared JSON for multi-player households/teams
14. **Full competitive history analysis** — season summary, deck performance by event

## DB Schema

```sql
-- Added to mtg_meta.db via db/event_hub_db.py:ensure_tables()

CREATE TABLE IF NOT EXISTS event_bookmarks (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id         TEXT UNIQUE NOT NULL,
    title            TEXT NOT NULL,
    store_name       TEXT,
    event_date       TEXT,           -- YYYY-MM-DD
    format_tags      TEXT,           -- JSON: ["modern", "regional_championship_qualifier"]
    event_type       TEXT,           -- rcq | store_championship | fnm | etc.
    entry_fee_cents  INTEGER,        -- in cents
    event_url        TEXT,
    lat              REAL,
    lng              REAL,
    dist_mi          REAL,
    status           TEXT DEFAULT 'interested',  -- interested | going | attended
    personal_notes   TEXT DEFAULT '',
    result           TEXT DEFAULT '',            -- "5-2 T8", "3-2", etc.
    deck_played      TEXT DEFAULT '',
    prize            TEXT DEFAULT '',
    added_at         TEXT DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS store_bookmarks (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    org_id      TEXT UNIQUE NOT NULL,
    name        TEXT NOT NULL,
    lat         REAL,
    lng         REAL,
    dist_mi     REAL,
    notes       TEXT DEFAULT '',
    added_at    TEXT DEFAULT CURRENT_TIMESTAMP
);
```

## Views

### Search View
- Zipcode + radius + type + format controls (same as current EventFinderTab)
- Results table with: Date, Distance, Store, Event, Entry, Format
- **Bookmark button** (star icon) on each row — adds to event_bookmarks
- **Bookmark Store** button on right-click context menu
- RCQ rows highlighted in accent color

### Calendar View
- Monthly grid: 7 columns (Sun-Sat), 5-6 week rows
- Day cells show: day number + colored event chips
- Chip colors:
  - RCQ: `#c0392b` (red)
  - Store Championship: `#2980b9` (blue)
  - Bookmarked events: gold border overlay
  - FNM/other: `theme.SURFACE` (gray)
- Max 3 chips per day; overflow shows "+N more"
- Prev/Next month navigation buttons
- Events source: last search results + all bookmarked events
- Click chip: shows event detail popover

### My Events View
- Table: Date | Distance | Store | Event | Status | Notes | Result | Deck
- Status column: dropdown (Interested / Going / Attended)
- Notes/Result/Deck: editable inline
- **Export .ics** button: generates iCalendar file for all Going+Interested events
- Filter by status: [All] [Interested] [Going] [Attended]
- Sort by date ascending (upcoming first)

### My Stores View
- Table: Store | Distance | Events This Month | Notes | Actions
- "Show Events" button: runs search filtered to this store's events
- Remove button
- Notes: editable inline

## .ics Export

Standard iCalendar format. Each bookmarked event = one VEVENT.
```
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//MTG Event Hub//EN
BEGIN:VEVENT
UID:{event_id}@mtg-event-hub
DTSTART:{YYYYMMDD}T120000Z
SUMMARY:{title} @ {store_name}
LOCATION:{store_name}
URL:{event_url}
DESCRIPTION:Entry: ${fee} | Format: {formats} | Status: {status}
END:VEVENT
...
END:VCALENDAR
```
